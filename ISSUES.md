## Priority Levels
-  <span style="color:red">**P0**</span> : Critical - Must fix immediately
- **P1**: High - Fix in current sprint
- **P2**: Medium - Fix in next sprint
- **P3**: Low - Fix when resources available

## Status Options
- **Not Started** - Issue created but work hasn't begun
- **In Progress** - Actively being worked on
- **In Review** - Complete but pending review
- **Blocked** - Waiting on external dependency
- **Completed** - Issue resolved and closed

## Instructions for Use
1. Add a new row for each issue with a unique Issue ID
2. Use P0-P3 for priority levels (P0 = highest priority)
3. Update Progress % as work progresses
4. Keep Status updated to reflect current state
5. Use YYYY-MM-DD format for dates
6. Mark as "Completed" in Status column when finished
7. Add any blockers or context in Notes column
